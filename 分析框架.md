## 一、数据获取策略

### 1.1 本地数据源（优先使用）

**股票行情数据：**
- **日线数据**: `E:\Code\export\stock\cache\{股票代码}_{股票名称}.csv`
- **周线数据**: `E:\Code\export\stock\weekcache\{股票代码}_{股票名称}.csv`
- **月线数据**: `E:\Code\export\stock\monthcache\{股票代码}_{股票名称}.csv`
- **数据范围**: 2001年至今，5790只股票
- **数据字段**: 日期、开盘、收盘、最高、最低、成交量、成交额(千元)、涨跌幅等

**其他本地数据：**
- 涨停主表：`G:\vibe\涨停数据汇总八字版.xlsx`（91765条，2020-2026）
- 个股八字：`G:\vibe\十二长生\个股八字完整版_优化.xlsx`（5482只）

### 1.2 网络数据源（本地数据缺失时使用）

**MCP搜索工具优先级（严格按顺序使用）：**

1. **智谱AI（zhipu）- 首选**
   - 工具：`mcp__zhipu__web_search`（搜索）、`mcp__zhipu__web_reader`（阅读）
   - 适用场景：中文信息、A股数据、中文财经资讯、行业动态
   - 优势：中文搜索效果最好，支持意图识别、时间范围过滤
   - 配置：已在`.mcp.json`中配置

2. **Tavily（tavily）- 次选**
   - 工具：`mcp__tavily__tavily_search`（搜索）、`mcp__tavily__tavily_extract`（提取）
   - 适用场景：英文信息、海外资讯、技术文档、国际市场信息
   - 优势：AI优化的搜索结果，支持深度搜索、包含AI生成答案
   - 配置：已在`.mcp.json`中配置

3. **直接URL读取 - 用户指定**
   - 工具：`mcp__zhipu__web_reader` 或 `mcp__tavily__tavily_extract`
   - 适用场景：用户提供具体网址时使用
   - 选择：中文网页用智谱，英文网页用Tavily

4. **Firecrawl - 最后备选**
   - 工具：`mcp__firecrawl__*`
   - 适用场景：需要深度爬取、PDF解析、复杂网页结构
   - 优势：深度网页内容抓取

**A级信源（优先）：**
- 巨潮资讯网 (cninfo.com.cn) - 公告、财报、招股书
- 上交所/深交所官网 - 龙虎榜、大宗交易、融资融券
- 东方财富 (eastmoney.com) - 行情、资金流、Choice数据
- 同花顺 (10jqka.com) - 行情、F10资料

**B级信源（辅助）：**
- Wind/万得 - 全品种行情、财务、估值（付费）
- 券商研报平台 - 慧博投研、萝卜投研
- 雪球 (xueqiu.com) - 市场情绪、投资者讨论

3. 数据质量检查

   ├─ 验证数据新鲜度
   └─ 标注数据截止日期

## 二、Skill 定位与核心理念

### 2.1 Skill 定位

### 2.1 Skill 定位

这是一个综合性的个股研究技能，整合了：
- **完整的Step 0-8分析流程**：从宏观定位到跟踪计划
- **产业链拆解与趋势验证**：供需格局、价格锚点、海外验证
- **资金面分析**：筹码分布、机构持仓、北向资金
- **技术面分析**：关键位置、量价配合、形态识别
- **期货关联分析**：大宗商品价格信号验证（适用时）
- **研报估值验证**：券商目标价合理性检验
- **三档目标价体系**：短中长期目标价测算
- **五维分析体系**：基本面+资金面+技术面+情绪面+事件催化
- **炒作风险识别**：19项快速筛选清单
- **可视化HTML输出**：红金蓝配色 + SVG图谱 + 目录导航

### 2.2 输出格式说明

**本skill直接输出HTML格式，不再输出Markdown。**

HTML报告特点：
- 红金蓝配色（符合中国股市文化：红涨）
- SVG产业链图谱可视化
- 悬停展开的目录导航
- 响应式设计，移动端友好
- 专业的数据可视化（评分条、情景卡片、时间线等）

### 2.3 核心理念

**1. 产业驱动，非情绪驱动**
- 顺的是产业趋势，不是K线趋势
- 产业逻辑是核心，技术面和资金面是辅助验证

**2. 先验证、后结论**
- 至少完成"趋势三要素"再下结论
- 基本面+资金面+技术面三重验证

**3. 先风控、后收益**
- 每份报告必须写清失效条件
- 明确止损位和逻辑破坏条件

**4. 盈亏比思维**
- 量化"向下X%空间 vs 向上Y%空间"
- 三档目标价体系（短期/中期/长期）

**5. 看大象比看蚂蚁容易**
- 优先研究信息透明度高的龙头
- 百亿以上市值优先，60亿以下慎重

**6. 低位谈逻辑，高位讲情绪**
- 位置决定分析重心
- 高位关注拥挤度和资金面

**7. 宁可错过不可做错**
- 等待业绩拐点确认，不急于抄底
- 炒作风险识别：区分"真成长"与"伪题材"

### 2.4 三条底线

1. **先事实、后推断**：关键结论必须有数据日期和来源
2. **先验证、后判断**：至少完成"趋势三要素"再下结论
3. **先风控、后收益**：每份报告必须写清失效条件

### 2.5 方法论定位

> `先市场定价 → 反推产业逻辑 → 回到公司业绩验证 → 交易与风控`

融合 `行业梳理/A股产业分析框架.md` 的自上而下产业方法论与"反向分析法"，形成完整闭环。
- **产业层面**（宏观→产业链→细分产品）：参考A股产业分析框架
- **个股层面**（公司筛选→弹性测算→估值风控）：按本文件Step 0-8执行

## 三、分析流程（Step 0 → Step 8）

### Step 0：任务锁定

| 字段 | 要求 |
|------|------|
| 标的 | 公司名、代码、交易所 |
| 周期 | `短线(1-4周)` / `中线(1-2季)` / `双周期` |
| 数据截止日 | 必须写绝对日期（YYYY-MM-DD） |
| 研究状态 | `首次覆盖` / `跟踪更新` / `事件复盘` |
| 风格预判 | `配置型` / `交易型` / `左侧博弈型` |

### Step 1：宏观与周期定位

#### 1a. 经济周期映射
- 经济阶段：`复苏前期` / `复苏中期` / `复苏后期` / `震荡`
- 公司所处周期与宏观的映射关系
- 政策阶段：`准备期` / `发力期` / `落地分化期`

**经济周期与板块对应：**

| 周期阶段 | 优势板块 | 逻辑 |
|---------|---------|------|
| 复苏前期 | 资源涨价（有色/稀土/化工） | 上游供给刚性，价格先行 |
| 复苏中期 | 设备/制造放量 | PMI回暖→机械设备订单 |
| 复苏后期 | 消费跟随 | 居民收入改善滞后传导 |

#### 1b. 政策与环境扫描
- 核心政策驱动（具体到政策名称、文件编号）
- 外部环境：汇率、地缘、海外需求、监管变化

#### 1c. 核心矛盾提炼（必做）

> 每只票必须用一句话提炼核心矛盾，格式：`XX vs YY`

**输出**：一句话回答"公司处在哪个顺风/逆风周期" + 核心矛盾

### Step 2：产业链深度拆解与趋势验证

#### 2a. 题材来源判断（必做，二选一）

| 方法 | 适用场景 | 示例 |
|------|---------|------|
| 消息面筛选法 | 冷门但逻辑硬的涨价/供给收缩方向 | 硫磺涨价 |
| 炒作脉络延伸法 | 主业确定性强，向新方向延伸 | 混动缸体→低空经济→机器人 |

筛选标准：**能否给具体公司带来真实业绩？** 不能则标注为"仅题材"。

#### 2b. 产业链图谱（必做，ASCII树形图）
```
产业方向：XXX
├── 上游（原材料/基础设施）
│   ├── 子项1 → 对公司影响
│   └── 子项2
├── 中游（产品/平台/解决方案）← 公司所在位置
│   ├── 业务线A（营收占比XX%，毛利率XX%）
│   ├── 业务线B
│   └── 业务线C
├── 下游（应用行业/客户）
│   ├── 客户A（占营收XX%）
│   └── 客户B
└── 关键判断
    ├── 判断1
    └── 判断2
```

#### 2c. 业务线拆解与趋势三要素（必做）

| 业务线 | 产业阶段 | 供需格局 | 价格锚点 | 海外验证 | 结论 |
|---|---|---|---|---|---|
| 主业A |  |  |  |  |  |
| 副业B |  |  |  |  |  |

**趋势三要素（每条业务线逐项打勾）：**

| 要素 | 确认方法 | ✅/❌ | 具体数据 |
|------|---------|------|---------|
| 供需格局 | 供给是否收缩/需求是否新增 |  |  |
| 价格锚点 | 现货/期货/ASP是否维持上行或高位 |  |  |
| 海外验证 | 海外龙头财报、供给变动是否同向 |  |  |

打分：`3/3` 趋势确认 / `2/3` 可跟踪待催化 / `<=1/3` 仅题材

#### 2d. 价值链利润分布分析（必做）

| 环节 | 代表公司 | 毛利率 | 净利率 | 利润趋势判断 |

## 三、信源分级与质量校验

### 3.1 信源分级（强制执行）

| 级别 | 来源 | 可否单独作结论 | 示例 |
|------|------|--------------|------|
| **A级** | 交易所公告、公司财报、官方部委/协会、统计局、海关、本地历史数据 | ✅ 可以 | 上交所公告、公司年报、工信部文件、本地CSV数据 |
| **B级** | 权威行情终端（Wind/Bloomberg）、主流财经媒体、可核验的调研纪要、东方财富/同花顺 | ⚠️ 需A级交叉验证 | Wind数据、财新、第一财经、东方财富 |
| **C级** | 券商观点、自媒体、论坛 | ❌ 不可单独下结论，仅作线索 | 券商研报、雪球、公众号 |

**本地数据优先原则：**
- 本地CSV数据（E:\Code\export\stock\cache\）属于A级信源
- 数据范围：2001年至今，5790只股票
- 优先使用本地数据，缺失时再使用WebSearch获取网络数据

**使用规则：**
1. **关键结论**（核心矛盾、趋势判断、投资建议）必须有 **≥2个独立来源** 支撑，其中至少 **1个A级**
2. **关键数字**（财务数据、价格、市值）必须标注 **来源+日期+单位**
3. **预测型信息**（业绩预测、价格预测）统一标注 `[推断]` 或 `[预测]`
4. **每份报告末尾** 必须附 **参考信息源清单**

### 3.2 数据新鲜度要求（强制）

| 数据类型 | 最大滞后期 | 备注 | 不符合时的处理 |
|---------|-----------|------|--------------|
| 行情数据（股价、市值） | T+0（当日） | 必须注明具体日期 | 标注"截至YYYY-MM-DD" |
| 财务数据（年报、季报） | 最新披露季度 | 超过1个季度需标注"待更新" | 标注"基于2025Q4数据，待更新" |
| 行业价格（现货、期货） | ≤7天 | 价格波动大的商品≤3天 | 标注"截至YYYY-MM-DD" |
| 产销数据（月度口径） | ≤1个月 | 行业协会数据 | 标注数据月份 |
| 政策文件 | 事件发生后T+1 | 必须引用原文或文号 | 附政策文件名称+发布日期 |
| 券商研报 | ≤3个月 | 超过3个月需重新验证 | 标注研报日期+机构名称 |

### 3.3 交叉验证规则（强制）

**必须交叉验证的内容：**
1. ✅ 核心矛盾判断
2. ✅ 趋势三要素结论
3. ✅ 业绩弹性测算的关键假设
4. ✅ 估值合理区间
5. ✅ 风险判断

**交叉验证方法：**
- **方法1：多源验证** - 同一结论由≥2个独立来源支撑
- **方法2：逻辑验证** - 上下游数据相互印证（如：上游涨价→中游成本上升）
- **方法3：时间验证** - 历史数据趋势支持当前判断

### 3.4 质量自检清单（分析完成后必做）

**信源质量检查：**
- [ ] 核心结论有≥2个独立来源支撑？

### 8.4 文件写入技术约束（重要！）

**问题背景：**
在生成分析报告时，AI模型可能遇到工具调用失败的问题，特别是Write工具的parameter标签内容为空，导致文件创建失败。

**强制执行规则：**

#### 规则1：优先使用Bash + cat heredoc方法
对于完整的分析报告（通常>100行），**必须使用Bash工具的cat heredoc方法**，不要使用Write工具。

```bash
cat > "路径/个股研究-公司名.md" << 'EOF'
# 完整的markdown内容
...
EOF
```

**原因：**
- Write工具在生成大量内容时，parameter标签内可能无法正常输出内容
- Bash + heredoc方法更稳定，适合大文件写入

#### 规则2：分段写入策略
如果报告内容特别长（>500行），采用分段追加策略：

```bash
# 第一段：创建文件头部
cat > "路径/文件.md" << 'EOF'
# 标题和前几个章节
EOF

# 第二段：追加中间内容
cat >> "路径/文件.md" << 'EOF'
## 中间章节
EOF

# 第三段：追加结尾
cat >> "路径/文件.md" << 'EOF'
## 最后章节
EOF
```

#### 规则3：JavaScript文件同样使用Bash方法
生成JS可视化文件时，也使用cat heredoc：

```bash
cat > "路径/viewer/stocks/公司名.js" << 'EOF'
window.stockData = window.stockData || {};
window.stockData['公司名'] = {
  // JSON数据
};
EOF
```

#### 规则4：禁止的错误模式
**❌ 绝对不要这样做：**
```xml
<!-- 错误：parameter标签为空 -->
<invoke name="Write">
<parameter name="file_path"></parameter>
<parameter name="content"></parameter>
</invoke>

<!-- 错误：parameter标签内没有实际文本 -->
<invoke name="Bash">
<parameter name="command"></parameter>
</invoke>
```

**✅ 正确做法：**
```xml
<!-- 正确：parameter标签内有实际的命令文本 -->
<invoke name="Bash">
<parameter name="command">cat > "文件路径" << 'EOF'
实际的文件内容...
EOF</parameter>
<parameter name="description">创建分析报告</parameter>
</invoke>
```

#### 规则5：执行前自检清单
在调用工具前，必须确认：
- [ ] parameter标签内有实际的文本内容（不是空标签）
- [ ] 使用了正确的工具（大文件用Bash，小文件可用Write）
- [ ] heredoc的EOF标记使用了单引号 `'EOF'`（避免变量替换）
- [ ] 文件路径使用了正确的格式（Windows下用正斜杠 `/` 或双引号包裹）

#### 规则6：失败后的处理
如果遇到工具调用失败：
1. **不要重复尝试同样的方法**
2. 立即切换到Bash + cat heredoc方法
3. 检查troubleshooting文档：`A股分析/troubleshooting-write-tool.md`

**参考文档：**
- 详细的故障排查指南：`A股分析/troubleshooting-write-tool.md`
- 包含错误示例、正确示例和解决方案

## 八、五维分析体系（补充）

### 9.1 五维分析框架

**权重分配：**
- 基本面（40%）：产业逻辑、业绩弹性、财务质量
- 资金面（20%）：筹码分布、主力资金、机构持仓
- 技术面（15%）：位置、趋势、量价配合
- 情绪面（15%）：市场热度、拥挤度、预期差
- 事件催化（10%）：政策、订单、产能投放

### 9.2 基本面分析（40%）

**核心要素：**
1. **产业逻辑（15%）**
   - 趋势三要素：供需格局、价格锚点、海外验证
   - 产业链位置：是否处于利润集中环节
   - 政策支持：是否有明确政策驱动

2. **业绩弹性（15%）**
   - 价格敏感度：涨价/降本对利润的影响
   - 量的弹性：产能利用率、新产能投放
   - 业绩确定性：订单可见度、客户稳定性

3. **财务质量（10%）**
   - 盈利能力：毛利率、净利率、ROE趋势
   - 现金流：经营性现金流/净利润比值
   - 资产负债：负债率、商誉、应收账款

**评分标准：**
- 优秀（32-40分）：产业趋势确认+业绩高增长+财务健康
- 良好（24-31分）：产业逻辑成立+业绩稳定增长
- 一般（16-23分）：产业逻辑待验证或业绩波动
- 较差（<16分）：产业逻辑不成立或财务恶化

### 9.3 资金面分析（20%）

**核心要素：**
1. **筹码分布（8%）**
   - 股东户数变化：集中度提升=看好
   - 机构持仓占比：>20%=机构认可
   - 北向资金持股：持续增持=外资看好

2. **主力资金（7%）**
   - 近20日主力净流入：持续流入=看好
   - 大单成交占比：>30%=主力活跃
   - 资金流向与股价背离：警惕信号

3. **融资融券（5%）**
   - 融资余额变化：增加=看多情绪
   - 融券余额：异常增加=做空力量
   - 融资买入占比：>10%=杠杆资金活跃

**评分标准：**
- 优秀（16-20分）：筹码集中+主力持续流入+融资活跃
- 良好（12-15分）：筹码稳定+主力净流入
- 一般（8-11分）：筹码分散或资金流向不明
- 较差（<8分）：筹码分散+主力流出+融券增加

### 9.4 技术面分析（15%）

**核心要素：**
1. **位置判断（6%）**
   - 相对历史高低点位置：低位>中位>高位
   - PE/PB历史分位：<30%=低估，>70%=高估
   - 均线系统：多头排列>震荡>空头排列

2. **趋势判断（5%）**
   - 趋势方向：上升趋势>震荡>下降趋势
   - 趋势强度：放量突破>缩量上涨>无量上涨
   - 关键位置：是否突破/跌破重要支撑阻力

3. **量价配合（4%）**
   - 放量上涨：健康
   - 缩量上涨：需警惕量能不足
   - 放量下跌：恐慌盘
   - 量价背离：警惕信号

**评分标准：**
- 优秀（12-15分）：低位+上升趋势+量价配合
- 良好（9-11分）：中位+趋势向上
- 一般（6-8分）：高位或趋势不明
- 较差（<6分）：高位+下降趋势+量价背离

### 9.5 情绪面分析（15%）

**核心要素：**
1. **市场热度（6%）**
   - 板块涨幅排名：前20%=热门
   - 换手率：>5%=活跃，>10%=过热
   - 涨停/跌停家数：板块情绪指标

2. **拥挤度（5%）**
   - 机构持仓占比：>30%=拥挤
   - 近期涨幅：月涨幅>30%需警惕
   - 市场关注度：研报数量、媒体报道

3. **预期差（4%）**
   - 当前处于预期传导哪个阶段
   - 市场已定价多少预期
   - 是否存在超预期空间

**评分标准：**
- 优秀（12-15分）：适度热度+低拥挤度+预期差大
- 良好（9-11分）：热度适中+拥挤度可控
- 一般（6-8分）：热度高+拥挤度上升
- 较差（<6分）：过热+高拥挤度+预期充分

### 9.6 事件催化（10%）

**核心要素：**
1. **政策催化（4%）**
   - 是否有明确政策支持
   - 政策落地时间窗口
   - 政策力度与持续性

2. **业绩催化（3%）**
   - 财报披露时间窗口
   - 业绩预告/快报超预期
   - 订单/合同公告

3. **事件催化（3%）**
   - 产能投放/新产品发布
   - 股权激励/股东增持
   - 行业会议/重大事件

**评分标准：**
- 优秀（8-10分）：多重催化+时间窗口明确
- 良好（6-7分）：有催化+时间窗口可预期
- 一般（4-5分）：催化不明确或时间窗口远
- 较差（<4分）：无催化或负面事件

### 9.7 五维综合评分

**总分计算：**
总分 = 基本面得分×40% + 资金面得分×20% + 技术面得分×15% + 情绪面得分×15% + 事件催化得分×10%

**评级标准：**
- A级（80-100分）：五维共振，强烈推荐
- B级（60-79分）：多维度支持，推荐关注
- C级（40-59分）：部分维度支持，谨慎参与
- D级（<40分）：多维度不支持，不建议参与

## 九、快速筛选与炒作风险识别

### 10.1 19项快速筛选清单

**使用场景：**
- 快速过滤明显不符合框架的标的
- 识别"伪题材"和高炒作风险标的
- 在深度分析前进行初步筛选

**筛选清单（任一项不通过需谨慎）：**

#### 第一层：基本面硬指标（5项）
1. [ ] **市值≥60亿**？（50亿以下信息不透明）
2. [ ] **主营业务清晰**？（不是多元化控股平台）
3. [ ] **最近一年扣非净利润为正**？（排除持续亏损）
4. [ ] **资产负债率<80%**？（排除高杠杆风险）
5. [ ] **无重大诉讼/处罚**？（近1年内）

#### 第二层：产业逻辑验证（5项）
6. [ ] **产业趋势可验证**？（有A级信源支撑）
7. [ ] **趋势三要素≥2/3**？（供需+价格+海外）
8. [ ] **3个季度内能落地业绩**？（不是远期概念）
9. [ ] **公司处于产业链关键环节**？（不是边缘参与者）
10. [ ] **主营业务与题材强相关**？（营收占比>30%）

#### 第三层：估值与位置（4项）
11. [ ] **PE/PB不在历史极高位**？（分位<90%）
12. [ ] **近3个月涨幅<100%**？（排除短期暴涨）
13. [ ] **盈亏比>1:1**？（向上空间>向下空间）
14. [ ] **有明确止损位**？（可量化的逻辑破坏条件）

#### 第四层：资金与交易结构（3项）
15. [ ] **换手率适中**？（日均3-8%，排除过冷/过热）
16. [ ] **机构持仓占比适中**？（10-40%，排除过度拥挤）
17. [ ] **主力资金近20日净流入**？（或至少不持续流出）

#### 第五层：风险排查（2项）
18. [ ] **无"不碰清单"特征**？（蹭概念/贸易商/主业拖累）
19. [ ] **无明显炒作痕迹**？（见下文炒作风险识别）

**通过标准：**
- 19项全通过：可进入深度分析
- 16-18项通过：需重点关注未通过项
- <16项通过：不建议深度分析

### 10.2 炒作风险识别（5个维度）

**维度1：基本面脱节度（20分）**

| 指标 | 低风险（16-20分） | 中风险（11-15分） | 高风险（≤10分） |
|------|-----------------|-----------------|----------------|
| 主营相关度 | 营收占比>50% | 营收占比20-50% | 营收占比<20%或无实际业务 |
| 业绩兑现时间 | 1-2季度内可验证 | 3-4季度内可验证 | >1年或无法验证 |
| 产业地位 | 细分龙头，市占率前三 | 行业参与者 | 边缘参与或蹭概念 |
| 订单/产能支撑 | 有明确订单或产能 | 有规划但未落地 | 无订单无产能 |

**维度2：估值泡沫度（20分）**

| 指标 | 低风险（16-20分） | 中风险（11-15分） | 高风险（≤10分） |
|------|-----------------|-----------------|----------------|
| PE历史分位 | <50% | 50-80% | >80%或亏损 |
| 近3月涨幅 | <30% | 30-80% | >80% |
| 与同业估值比 | 溢价<30% | 溢价30-80% | 溢价>80% |
| PEG | <1 | 1-2 | >2或无法计算 |

**维度3：资金投机度（20分）**

| 指标 | 低风险（16-20分） | 中风险（11-15分） | 高风险（≤10分） |
|------|-----------------|-----------------|----------------|
| 换手率 | 日均3-8% | 日均8-15% | 日均>15% |
| 龙虎榜频率 | 近1月0-1次 | 近1月2-3次 | 近1月>3次 |
| 游资占比 | 龙虎榜游资<30% | 龙虎榜游资30-60% | 龙虎榜游资>60% |
| 融资买入占比 | <5% | 5-10% | >10% |

**维度4：舆情炒作度（20分）**

| 指标 | 低风险（16-20分） | 中风险（11-15分） | 高风险（≤10分） |
|------|-----------------|-----------------|----------------|
| 股吧/社区热度 | 讨论适中，理性分析 | 热度高，情绪化增加 | 刷屏级热度，极度情绪化 |
| 自媒体推荐 | 少量专业分析 | 多个自媒体推荐 | 大量自媒体集中推荐 |
| 概念标签数量 | 1-2个核心概念 | 3-4个概念 | >5个概念（万金油） |
| 公司互动易回复 | 谨慎客观 | 模糊暗示 | 主动蹭热点 |

**维度5：技术形态风险（20分）**

| 指标 | 低风险（16-20分） | 中风险（11-15分） | 高风险（≤10分） |
|------|-----------------|-----------------|----------------|
| 涨停板数量 | 近1月0-2个 | 近1月3-5个 | 近1月>5个 |
| 量价配合 | 放量上涨，健康 | 缩量上涨或量价背离 | 天量滞涨或巨量跌停 |
| 技术位置 | 低位或中位 | 中高位 | 历史高位或加速赶顶 |
| K线形态 | 稳步上涨 | 快速拉升 | 连续一字板或T字板 |

**炒作风险综合评分：**
- 80-100分：低炒作风险，可正常分析
- 60-79分：中等炒作风险，需警惕情绪退潮
- 40-59分：高炒作风险，不建议追高
- <40分：极高炒作风险，远离

### 10.3 "真成长" vs "伪题材" 识别

**真成长特征：**
1. ✅ 产业逻辑有A级信源支撑（政策文件/行业数据/龙头财报）
2. ✅ 公司有实际订单/产能/技术储备
3. ✅ 业绩弹性可量化（价格敏感度公式）
4. ✅ 1-2个季度内可通过财报验证
5. ✅ 估值虽高但有业绩增长支撑（PEG<2）
6. ✅ 机构持仓为主，游资占比低

**伪题材特征：**
1. ❌ 产业逻辑仅有C级信源（自媒体/论坛传闻）
2. ❌ 公司仅有"规划"/"研发中"，无实际业务
3. ❌ 业绩弹性无法量化或逻辑链条断裂
4. ❌ 业绩兑现时间>1年或根本无法兑现
5. ❌ 估值极高且无业绩支撑（PE>100或亏损）
6. ❌ 游资主导，龙虎榜频繁出现

**典型案例对比：**

| 维度 | 真成长案例 | 伪题材案例 |
|------|----------|----------|
| 产业逻辑 | AI算力需求爆发（英伟达财报+国内政策） | 元宇宙概念（仅有媒体炒作） |
| 公司参与度 | 液冷服务器订单落地，产能扩张 | 仅在互动易表示"有关注" |
| 业绩验证 | 下季度财报可验证收入增长 | 无明确业绩兑现路径 |
| 估值 | PE 40倍，PEG 0.8 | PE 150倍或亏损 |
| 资金结构 | 机构持仓40%，北向增持 | 游资龙虎榜占70% |
| 股价表现 | 稳步上涨，回调有支撑 | 连续涨停后闪崩 |

### 10.4 快速筛选工作流

**Step 1：初筛（5分钟）**
1. 查看市值、主营业务、最近财报
2. 检查是否有重大诉讼/处罚
3. 通过→进入Step 2；不通过→放弃

**Step 2：产业逻辑验证（10分钟）**
1. 搜索产业趋势的A级信源
2. 验证公司是否有实际业务参与
3. 评估业绩兑现时间窗口
4. 通过→进入Step 3；不通过→标注"仅题材"

**Step 3：炒作风险评估（5分钟）**
1. 查看近3月涨幅、换手率、龙虎榜
2. 检查股吧/社区热度
3. 计算炒作风险评分
4. 低/中风险→进入深度分析；高风险→放弃或等待回调

**Step 4：深度分析（按Step 0-8执行）**
- 完整执行本框架的Step 0-8流程
- 输出完整研究报告

## 十、完整分析工作流（整合版）

### 11.1 标准工作流（首次覆盖）

**阶段1：快速筛选（20分钟）**
1. 执行19项快速筛选清单
2. 炒作风险识别（5个维度）
3. 判断：真成长 vs 伪题材
4. 决策：进入深度分析 / 放弃 / 观察

**阶段2：基础研究（1-2小时）**
1. Step 0：任务锁定
2. Step 1：宏观与周期定位
3. Step 2：产业链深度拆解
4. Step 3：公司筛选与质量评分
5. 决策：继续深入 / 降级为"仅观察"

**阶段3：深度分析（2-3小时）**
1. Step 4：业绩弹性测算
2. Step 5：风险分析与逻辑破坏条件
3. Step 6：估值-赔率与买卖时机
   - 资金面分析
   - 技术面分析
   - 期货关联分析（如适用）
   - 研报估值验证
   - 三档目标价体系
4. Step 7：对标与对比分析
5. Step 8：跟踪计划与综合结论

**阶段4：五维综合评分（30分钟）**
1. 基本面评分（40%）
2. 资金面评分（20%）
3. 技术面评分（15%）
4. 情绪面评分（15%）
5. 事件催化评分（10%）
6. 计算总分，确定评级

**阶段5：输出与复核（30分钟）**
1. 撰写完整研究报告（Markdown格式）
2. 执行质量自检清单
3. 生成JavaScript可视化文件
4. 附参考信息源清单

### 11.2 跟踪更新工作流

**高频跟踪（周度，15分钟）**
1. 更新产品现货价格（如适用）
2. 查看资金流向变化
3. 关注行业动态/公司公告
4. 判断：逻辑是否变化？

**季度跟踪（财报发布后，1小时）**
1. 更新财务数据（营收/净利/毛利率/现金流）
2. 验证业绩弹性测算是否准确
3. 更新三档目标价
4. 调整质量评分和五维评分

**事件驱动跟踪（事件发生后，30分钟）**
1. 验证事件真实性（信源级别）
2. 评估对核心逻辑的影响
3. 更新逻辑破坏条件
4. 决策：继续持有 / 加仓 / 减仓 / 离场

### 11.3 决策树

```
标的筛选

    └── 事件驱动 → 决策：持有/加仓/减仓/离场
```

### 案例：某AI算力设备公司

**快速筛选（通过）：**
- ✅ 市值120亿
- ✅ 主营：AI服务器液冷散热设备
- ✅ 2025年扣非净利润3.2亿（+85%）
- ✅ 资产负债率45%
- ✅ 无重大诉讼

**炒作风险识别：75分（中等风险）**
- 基本面脱节度：18分（订单落地，产能扩张）
- 估值泡沫度：12分（PE 45倍，历史70%分位）
- 资金投机度：14分（日均换手8%，龙虎榜2次/月）
- 舆情炒作度：16分（热度适中）
- 技术形态风险：15分（近1月涨幅60%）

**核心矛盾：**
AI算力需求爆发 vs 液冷渗透率提升速度

**趋势三要素：3/3**
- ✅ 供需格局：AI服务器出货量高增长，液冷渗透率从5%→20%
- ✅ 价格锚点：单台液冷设备价值量10-15万，价格稳定
- ✅ 海外验证：英伟达GB200要求液冷，海外数据中心加速改造

**业绩弹性：**
每新增1000台液冷设备 → 年增收入1.2亿 → 年增净利润0.36亿

**三档目标价：**
- 短期（3个月）：35元（2025年PE 40倍）
- 中期（12个月）：50元（2026年PE 35倍）
- 长期（3年）：80元（2028年PE 30倍）

**五维评分：82分（A级）**
- 基本面：35/40（产业逻辑硬+业绩高增长）
- 资金面：16/20（机构持仓35%，北向增持）
- 技术面：11/15（中高位，趋势向上）
- 情绪面：12/15（热度适中，拥挤度可控）
- 事件催化：8/10（英伟达GB200放量+国内政策）

**综合结论：**
- 判断：当前更像"趋势"（产业逻辑硬+业绩兑现中）
- 风险等级：中（估值偏高，需警惕情绪退潮）
- 风格标签：交易型（景气成长）
- 操作建议：择时参与，回调至30元以下可考虑配置

## 附录C：产业链拆解实战案例

### C.1 新能源车产业链利润分配演变

```
2020-2021（产能紧缺期）：
上游原材料（碳酸锂/六氟磷酸锂）毛利率70-80% → 利润集中在资源端
中游电池厂（宁德时代）毛利率25-30% → 被上下游挤压
下游整车厂（比亚迪/特斯拉）毛利率15-20% → 涨价传导受限

2023-2025（产能过剩期）：
上游原材料毛利率暴跌至10-20% → 利润向中下游迁移
中游电池厂毛利率回升至30-35% → 规模效应+技术溢价
下游整车厂毛利率分化：
  - 高端品牌（理想/问界）毛利率20-25% → 品牌溢价
  - 低端品牌（零跑/哪吒）毛利率<10% → 价格战受损

投资启示：
- 产能紧缺期买上游资源（赚供需错配的钱）
- 产能过剩期买中游龙头（赚规模效应+技术壁垒的钱）
- 全周期避开低端整车厂（两头受挤，无定价权）
```

### C.2 AI算力产业链利润分配

```
完整传导链：
英伟达GPU（毛利率70%+）
  ↓ 成本占比：服务器总成本的40-50%
AI服务器整机（浪潮/超微，毛利率15-20%）
  ↓ 成本占比：各零部件成本占比
├─ 电源模组（麦格米特，毛利率35%，单台价值>10万）
├─ 高速连接器（中航光电，毛利率40%，国产化率<15%）
├─ PCB主板（沪电股份，毛利率25%，价值量提升5-10倍）
├─ 液冷散热（英维克，毛利率30%+）
└─ MLCC（风华高科，毛利率20%，单台用量万颗级）

利润分配格局：
- 利润最集中：英伟达（垄断地位，70%+毛利率）
- 利润次集中：核心零部件（电源/连接器/液冷，30-40%毛利率）
- 利润最薄：整机组装（浪潮/超微，15-20%毛利率）

投资优先级排序：
1. 高壁垒零部件（电源/连接器/液冷）：毛利率高+供给紧+下游对价格不敏感
2. 英伟达产业链受益者（PCB/覆铜板）：确定性强+量价齐升
3. 整机厂：规模大但利润薄，适合指数配置而非个股深挖
```

### B1. 数据优先级

**其他用户环境：**

### B2. 数据获取流程

```
Step 1: 检查本地CSV缓存
  ├─ 存在 → 读取日线数据
  └─ 不存在 → 使用AkShare获取

Step 2: 补充AkShare数据
  ├─ 实时行情（价格、市值、换手率）
  ├─ 财务数据（营收、利润、ROE）
  ├─ 资金流向（主力净流入）
  ├─ 技术指标（MA5/MA20/RSI）
  └─ 52周高低点

Step 3: 可选MCP搜索
  ├─ 检查MCP配置
  ├─ 存在 → 获取行业分析、产业链研究
  └─ 不存在 → 跳过，使用基础数据

Step 4: 生成JSON缓存
  └─ 保存到 output/{股票名称}_技术分析.json
```

**配置文件：** `.mcp.json` 或 `~/.claude/settings.json`

**示例配置：** 参见 `.mcp.json.example`

**详细指南：** 参见 `MCP配置指南.md`
